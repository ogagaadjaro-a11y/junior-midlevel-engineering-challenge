# Mini System Design: Transaction Monitoring Dashboard

## Timebox

24 minutes.

## Scenario

Design a simple transaction monitoring dashboard for an internal fintech operations team.

The dashboard should allow users to view transactions, search by reference, filter by status, view transaction details, see loading and error states, and refresh data.

## What To Cover

Explain:

- Main components.
- State management approach.
- Data flow.
- API integration approach.
- Loading and error handling.
- Empty states.
- Basic accessibility considerations.
- How you would test the UI.

## Expected Output

You may answer with a component diagram, bullet points, pseudocode, or a short written design. Keep the design practical for a small internal dashboard.

## Evaluation Focus

The interviewer is looking for clear component boundaries, reasonable state management, practical data flow, user-focused edge cases, accessibility awareness, and test thinking.
