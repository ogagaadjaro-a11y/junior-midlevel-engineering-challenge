# Mini System Design: Wallet Transaction API

## Timebox

24 minutes.

## Scenario

Design a simple wallet transaction API for a fintech product. The API should allow users to create wallets, fund wallets, transfer money between wallets, view balances, and view transaction history.

Keep the design practical. You do not need to design a large distributed system or production banking platform.

## What To Cover

Explain:

- Main API endpoints.
- Request and response formats.
- Basic database tables or data models.
- How invalid transfers are prevented.
- How insufficient balance is handled.
- How duplicate transfers are avoided.
- How meaningful errors are returned.
- How you would test the system.

## Expected Output

You may answer using diagrams, bullet points, pseudocode, or a short written design. Focus on clarity and tradeoffs.

## Evaluation Focus

The interviewer is looking for practical API thinking, clean data modeling, validation, error handling, idempotency awareness, and a testing mindset.
